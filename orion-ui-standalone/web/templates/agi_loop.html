Anatomy of a Tick

Each tick is a bounded mini-loop. The agent must respond with structured JSON step objects.

1. Prompt Assembly

Base system prompt + FAISS memory injection + directive soul-scripts + always-on notes + working set state

2. Step Loop

Up to max_steps_per_tick LLM calls. Each outputs JSON: think, tool, or stop

3. Tool Dispatch

Max max_tool_calls_per_tick invocations. Boundary logger enforces safety. Stasis blocks all non-runtime_info

4. Memory Flush

Proposed memories collected across steps are bulk-persisted to FAISS vault at end-of-tick with scope, category, and tags

5. Metering & Journaling

Token usage, estimated cost, step summaries logged. NarrativeWriter creates human-readable diary entries

6. Outcome Report

Each tick returns a TickOutcome: steps, tools, memories, errors, cost â€” aggregated at burst level

Escalation Logic
Routing Algorithm
  1. Classify task type from metadata
  2. Start at lowest tier that handles the task class
  3. If verification fails, retry once at same tier
  4. If stuck (same error 3+ times), escalate one tier
  5. Enforce hard caps on iterations and tokens per tier
  6. Budget gates refuse escalation if budget exceeded
